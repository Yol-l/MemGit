一、AI 记忆的基本问题

大模型先天无状态——每次对话都是"初次见面"。它不记得你是谁、你们聊过什么。

挑战 1

上下文窗口有限

就算最新的模型也就 200K token 左右，对话一长就把早期的记忆挤掉了。

挑战 2

AI 有幻觉

记住的东西可能是错的，而且会自信地复述错误信息。

挑战 3

跨会话空白

每次新建对话，AI 对你的了解从头开始，没有连续性。

二、当前市面上的三条技术路线

路线一

外置记忆库

最成熟，商业化最充分

不改变模型，在外面建数据库存记忆。对话时检索注入上下文。

代表：Mem0、Letta、Graphiti

路线二

参数化记忆

效果最好，成本最高

把知识微调进模型权重，变成"肌肉记忆"——不用检索就能用。

代表：Character.AI、Gorilla

路线三

隐式记忆

最新，效率最高

不存具体内容，存"思考时的脑活动模式"——KV Cache 管理。

代表：Google CoMEM、LongBench

个人关注方向：三条路线不互斥。我感兴趣的是用 Git + 纯文本这种轻量方式来做记忆管理——关键不在于技术多先进，而在于数据透明、用户可控、跨平台通用。

三、我感兴趣的方向：用 Git 管理记忆

Git 做版本管理、Markdown 做记忆载体、纯本地不依赖云。不是我发明的，是一种小众但有意思的实践方向。

方案存储我的关注点
Mem0向量数据库检索靠向量相似度，AI 自己判断哪条记忆有用——有点重
Letta(MemGPT)三层内存系统设计很好，但跟平台绑定，不跨工具
Graphiti时序知识图谱复杂，图谱结构对 AI 不如 Markdown 直观
ChatGPT 记忆服务端数据库黑箱，用户看不到记了什么、改不了、拿不走
Git + SkillsMarkdown + Git我在尝试的方向——透明、轻量、你拥有数据
我的兴趣点：探索一套"用户拥有数据、AI 自成长、可跨平台"的记忆方案。不一定是最好的，但我想试试。

四、MemGit 的设计思路

用 Git 和 Markdown 做记忆——你拥有数据，AI 自我维护。

├── IDENTITY.md # 你是谁（核心身份） ├── INDEX.md # 记忆导航图 ├── knowledge/ # 个人记忆（不限文件） │ ├── events.md # 近期持续性事件 │ ├── principles.md # 价值观与原则 │ ├── habits.md # 习惯与风格 │ ├── learning.md # 学习成长 │ └── goals.md # 人生目标 ├── projects/ # 项目记忆（一个项目一个文件） ├── strategies/ # 记忆维护策略（可共享） └── library/ # 代码/资料备忘（按需读）

纯文本 · 纯本地 · 纯 Git · 任意平台通用

五、核心原则：自成长 + 异常恢复

自成长

系统自己演化架构、新增分类、自我优化。

包含：渐进自检、养护优先、主动感知、膨胀合并

异常恢复

AI 幻觉自动纠偏、记忆损坏恢复、规则冲突自检。

用户发现错误只需一句话，系统自动修正

规则在先

先定规则再建文件。AI 不能自由发挥。

每条记忆写入前经过 8 条硬边界自检

六、每轮对话的记忆流转

用户说话
→
过记忆
→
自然接话
→
记不记
→
自检 8 条
→
写入 + 审计
📥 读取：每次启动读身份 → 事件 → 索引 → 渐进按需深入
📝 写入：归属判断 → 打开文件读 write_rules → 按规则写 → 自检 8 条 → 膨胀阈值检查
🔄 自愈：用户否认 → 立即修正 → 同级降级 → 不再引用，直到用户重新确认
📊 审计：每次会话结束输出变更摘要，写入 git commit。下次启动自检校验一致性

七、系统自闭环

闭环机制频率
记忆闭环写入 → 自检 → 修复 → 审计 → 下次验证每会话
自成长闭环系统反思 4 维度 → 信号积累 → 达阈值 → 自适应协议 → CHANGELOG 记录每 5 轮
异常恢复闭环用户否认 → 修正 + 降级 → 不再引用 → 自然话题确认用户触发
规则闭环观察违规模式 → 提议改规则 → 用户确认 → 归档旧规则约 30 次会话/次
膨胀合并闭环H2 超 30 行 → 立即压缩 → 存档超 50 行 → 二次压缩P0 立即触发
系统的五种闭环相互嵌套，高频的（每会话）保证内容质量，低频的（30 会话/次）保证规则质量。系统不依赖外部维护，自己检查、自己修复、自己进化。

八、设计中想实现的自愈机制

# 假设场景：AI 记错了
AI: "你之前说喜欢榴莲"
你: "我不喜欢"
→ 设计上希望它自动修正 + 不再提错误的记忆

# 这就是为什么在探索自愈机制

这个方向还在探索中——目标是在正常对话中自然纠正记忆，用户不需要主动"修改"，说一句就行。

十一、设计理念总结

记忆是待验证的断言，不是事实
AI 会错，但每轮对话都是修复机会
用户拥有数据，AI 提供维护
纯文本 + Git，数据是你的，AI 只是帮你管理
自成长 > 完美初始设计
不追求一次性设计完备，系统自己演化和修复
不替代用户决策，不骚扰用户
AI 只能提议，不能改规则。用户否认才处理
