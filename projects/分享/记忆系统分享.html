AI 记忆的基本问题

大模型先天无状态——每次对话都是"初次见面"。它不记得你是谁。

▦
挑战 1

上下文窗口有限

最先进的模型也只有约 200K token——对话一长就把早期的信息挤掉了。

⚠
挑战 2

AI 有幻觉

记住的内容可能完全是捏造的，而且会自信地复述，用户难以察觉。

↻
挑战 3

跨会话空白

每次新对话从头开始，没有连续性。AI 无法积累对用户的长期理解。

当前三条技术路线

路线一

外置记忆库

最成熟，商业化最充分

不改变模型，在外面建数据库。对话时检索并注入上下文。像外接硬盘。

Mem0 · Letta · Graphiti

路线二

参数化记忆

效果最好，成本最高

把知识微调进模型权重，让它变成"肌肉记忆"。不需要检索就能用。

Character.AI · Gorilla

路线三

隐式记忆

最新，效率最高

不存具体内容，存"思考时的脑活动模式"——KV Cache 智能管理。

Google CoMEM

趋势：三条路线不互斥。混合架构正在成为主流——参数化（高频）+ 外置（历史）+ 隐式（当前）。

我感兴趣的方向：用 Git 管理记忆

轻量、透明、用户拥有数据——这是一个正在探索的个人项目。

方案存储方式我的关注点 Mem0向量数据库检索靠相似度，AI 无法自行判断哪条有用——有些重 Letta (MemGPT)三层内存架构系统设计优秀，但与平台绑定，不跨工具 ChatGPT 记忆服务端黑箱用户看不到、改不了、拿不走 Git + SkillsMarkdown + Git透明、轻量、用户拥有完整数据控制权——我在尝试的方向
探索：一套"用户拥有数据、AI 自我维护、跨平台通用"的记忆方案。不一定是最好的，但值得一试。

MemGit 的设计：文件即记忆

memories/ ├── IDENTITY.md 你是谁 ├── INDEX.md 记忆导航图 ├── knowledge/ 个人记忆（不限文件） │ ├── events.md 近期事件（时间段） │ ├── principles.md 价值观与原则 │ ├── learning.md 学习成长 │ └── goals.md 人生目标 ├── projects/ 项目记忆（一个项目一个目录） │ ├── memgit/ MemGit 系统建设 │ └── 分享/ 部门技术分享 ├── strategies/ 记忆维护策略（可共享） └── library/ 代码资料（按需读）

纯文本 · 纯本地 · 任意平台通用 · Git 版本控制

核心原则：自成长 + 异常恢复

此系统的存在意义。不可修改，后续一切围绕它们展开。

自成长

系统自己演化架构、新增分类、自我优化效率。渐进自检、养护优先、主动感知、膨胀合并。

异常恢复

AI 幻觉自动纠偏、记忆损坏恢复、规则冲突自行检测。用户说一句即可修正。

只记长期

6 个月后还有价值的信息才进入记忆系统。

规则在先

先定规则再建文件。每条写入经过 8 条硬边界自检。

主动感知

根据记忆主动给建议、关联方案，根据情绪调整话术。交互越多成长越快。

记忆可纠

每轮对话都是校验机会。用户否认立即修正，同级降级不再引用。

每轮对话的五步流转

用户发言
→
过记忆
→
自然接话
→
记不记
→
自检 + 审计

读取

每次启动：身份 → 事件 → 索引 → 渐进按需深入。只读必要的。

写入

归属判断 → 读文件 write_rules → 按规则写 → 8 条硬边界自检 → 阈值检查。

自愈

用户否认 → 立即修正 + 同级降级 → 不再引用 → 自然话题确认。

审计

每次会话结束：变更摘要 + git commit + push。下次启动自检校验一致性。

系统自闭环

闭环机制频率 记忆闭环写入 → 自检 → 修复 → 审计 → 下次验证每会话 自成长闭环系统反思 4 维度 → 信号积累 → 达阈值 → 自适应协议每 5 轮 异常恢复闭环用户否认 → 修正 + 降级 → 不再引用 → 自然确认用户触发 规则闭环观察违规 → 提议改规则 → 用户确认 → 归档旧规则30 会话/次 膨胀合并闭环H2 超 30 行 → 压缩 → 存档超 50 行 → 二次压缩P0 即时
五个闭环嵌套运行。高频的保证内容质量，低频的保证规则质量。系统不依赖外部维护，自己检查、自己修复、自己进化。

设计中想实现的自愈机制

发现错误记忆

当 AI 引用了错误记忆，用户只需说"我没有这样说"或"这个不对"。

系统自动处理

AI 立即修正该条目，同时将同一批次写入的记忆降级为"存疑"。

降级不再引用

存疑的记忆不再出现在主动建议中，只等用户自己重新提起。

自然恢复

用户在后续对话中自然确认正确版本 → 升级为"确认的"，恢复正常使用。

不需要用户主动"修改"——正常对话中自然纠正，每轮对话都是修复机会。
