AI 记忆的基本问题

大模型先天无状态——每次对话都是"初次见面"。它不记得你是谁、你们聊过什么。

▦
挑战 1

上下文窗口有限

最先进的模型也只有约 200K token。对话一长，早期信息就被挤掉了——像只能记住最近 20 分钟对话的人。

⚠
挑战 2

AI 有幻觉

记住的内容可能是捏造的——而且会自信地复述。用户很难察觉，错误信息会持续污染后续对话。

↻
挑战 3

跨会话空白

每次新对话从零开始。一个星期后回来，AI 连你叫什么都不知道。没有连续性。

当前三条技术路线

路线一

外置记忆库

最成熟，商业化最充分

不改变模型，在外面建数据库。对话时检索并注入上下文，像外接硬盘。

代表：Mem0 · Letta · Graphiti

路线二

参数化记忆

效果最好，成本最高

把知识微调进模型权重，像"肌肉记忆"——不需要检索就能用。

代表：Character.AI · Gorilla

路线三

隐式记忆

最新，效率最高

不存具体内容，存"思考时的脑活动模式"——KV Cache 智能管理。

代表：Google CoMEM

趋势：三条路线不互斥。混合架构正在成为主流——参数化（高频）+ 外置（历史）+ 隐式（当前）。

我感兴趣的方向：Git + 纯文本管理记忆

轻量、透明、用户拥有数据——正在探索的个人项目。

方案存储方式我的观察 Mem0向量数据库检索靠相似度——AI 没法自行判断哪条记忆在当前语境下最有用 Letta (MemGPT)三层内存架构系统设计优秀，但与平台绑定，不能跨工具使用 ChatGPT 记忆服务端黑箱用户看不到记了什么、改不了、拿不走 Git + SkillsMarkdown + Git透明、轻量、用户拥有完整数据控制权——我在尝试的方向

系统结构：文件即记忆

memories/ ├── IDENTITY.md 你是谁 ├── INDEX.md 记忆导航图 ├── knowledge/ 个人记忆（不限文件） │ ├── events.md 近期事件 │ ├── principles.md 价值观与原则 │ ├── habits.md 习惯与风格 │ ├── learning.md 学习成长 │ └── goals.md 人生目标 ├── projects/ 项目记忆（一个项目一个目录） │ ├── memgit/ MemGit 建设 │ └── 分享/ 本次部门分享 └── strategies/ 记忆维护策略（可共享）

纯文本 · 无平台依赖 · Git 版本控制 · 任意 AI 工具通用

核心原则

系统的存在意义——不可修改，后续一切围绕它们展开。

自成长

系统自己演化架构、新增分类、自我优化效率。渐进自检、养护优先、主动感知、膨胀合并。

异常恢复

AI 幻觉自动纠偏、记忆损坏恢复、规则冲突自行检测。用户说一句即可修正。

只记长期

6 个月后还有价值的信息才进入记忆系统。

规则在先

先定规则再建文件。每条写入经过 8 条硬边界自检。AI 不能自由发挥。

主动感知

根据记忆主动给建议、关联方案，根据情绪调整话术。交互越多成长越快。

记忆可纠

每轮对话都是校验机会。用户否认立即修正，同级降级不再引用。

读取：启动

身份 → 事件 → 索引 → 渐进按需深入。只读必要的，不塞满上下文。

写入：每轮

归属判断 → 读 write_rules → 按规则写 → 8 条自检 → 阈值检查。

自愈：随时

用户否认 → 修正 + 降级 → 不再引用 → 自然确认恢复。

审计：收尾

每次会话结束输出变更摘要，写入 git commit。下次启动校验一致性。
